Blog... Experiments with MongoDB

This week I did 5 experiments using MongoDB.

Display data from MongoDB

This week is similar to last week's experiments, except that instead of using static data, I have used MongoDB to fetch and insert data. In this experiment, I have inserted some book information into the database and retrieved them and displayed it in a table.

Delete a Book from the Database

This experiment is an extension of the previous experiment where I have created a new button to delete a book from the database. On clicking the delete button, the record for that book is deleted from the database.

Insert new book to the database

This experiment is an extension of the previous experiment where I have added 2 other buttons: one to add a new book and another to select and edit a particular book. When the edit button is clicked, it's value is copied onto the form, this is known as copy by reference.

Adding Chapters for a book

This experiment is an extension of the previous experiment where I have added a new button to add a chapter for a book when it is selected. For a selected book, it's chapters are displayed in a table. When the add button is clicked for a particular chapter, the chapters array of that Book is updated in the database.

Deleting chapter for a book

This experiment is an extension of the previous experiment where I have added new buttons to delete each chapter of a book when it is selected. When the delete button is clicked for a particular chapter, it's index is obtained and spliced from the nested chapters list.

Summary

This week was very taxing, but interesting. I started with NodeJMongoDB from the scratch with guidance from Prof Jose's lectures. It was time consuming to understand the architecture and re-implement the experiments that were demonstrated in class. I faced some issues while installing mongoose which took up most of my time. Apart from that, I learnt several new concepts.
