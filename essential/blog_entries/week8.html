Blog... Experiments with MongoDB

This week I did 5 experiments using PassportJS.

Setting up PassportJS and Navbar

In this experiment, I have setup passportJS and built a NavBar. Passport.js is a flexible authentication middleware that allows users to log in and can work with connect/express. It is flexible as it allows for different authentication strategies (Twitter, Facebook or your own local database and so on) and it allows us to specify any route or output during authentication.

Login

This experiment is an extension of the previous experiment where I have added a login functionality. The usernames and passwords are stored in the UserModel which is retrieved at the time of login and authenticated using the LocalStrategy.

Restricting Profile to authenticated users

In this experiment, I restricted the Profile page to only authenticated logged-in users. To do this, I resolve the route only when a check is made to see if the user is currently logged in.

Logout

This experiment is an extension of the previous experiment where I have added a logout functionality for a user to logout. This was fairly simple using PassportJS.

Register a new User

In this experiment, I have added the functionality to register a new user. When the username and passwords are entered, a check is made to check if the passwords match and a newUser object is created and the information is pushed on to the database. Once the user registers, it will be redirected to the user's profile page.

Summary

This week was very interesting. I decided my project and started working on the design. Meanwhile, I experimented with PassportJS which I will be using in my Project as well. Overall, a productive week.
