Profile Page



The Profile Page is displayed after an authenticated user is logged in. I have restricted the Profile page only to authenticated logged-in users. To do this, the route is resolved only after a check is made to see if the user is currently logged in. For example, if you try to access the Profile url without logging in, an alert will be displayed and automatically redirect you to the Login page. The profile page contains the user's information like the user's account information, posts and the list of users that the user is following. The navigation menu now shows the Profile tab, a button to add a new Post and a Logout button.
