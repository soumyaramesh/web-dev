jQuery UI Accordion

About the Experiment

This experiment demonstrates the accordion widget of jQuery UI which displays headers with collapsible content panels for presenting information in a limited amount of space. This is a more sophisticated way of representing details and summary based on the user's choice. The accordion widget also provides several options which are discussed below.

Code Snippet:

$(document).ready(function() {
        $("#accordion").accordion({
        active: 1,
        heightStyle: "content",
        collapsible: true
        });
    });

Description

#accordion is the id for each header. $("#accordion").accordion is used to reference the headers with id="accordion" and .accordion is used to convert a pair of headers and content panels into an accordion. We can specify several options such as active, animate, collapsible, event, header, heighStyle and so on. Here, I have experimented with active, heighStyle and collapsible.
  • active is used to specify which header should be active when the accordion is loaded. Since I have specified active:1, the second panel is active. 0 is for the first panel.
  • heighStyle has 3 options. "content" is used when the height of the panel is according to that of the content size. "auto" is used when all panels will be set to the height of the tallest panel and "fill" is used to expand to the available height based on the accordion's parent height.
  • collapsible is used when all the sections can be closed at once. It allows to collapse the active section, which is not possible in the default setting where at least one panel will be active.
