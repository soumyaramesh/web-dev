About the Experiment

This experiment is an extension of the previous experiment where I have added 2 other buttons: one to add a new book and another to select and edit a particular book. When the edit button is clicked, it's value is copied onto the form, this is known as copy by reference.

The figure below shows the contents of the database when a new book called Fountainhead is added to the existing database.


Code Snippet:

        
    

server.js
app.post("/book", function (req, res) {
    var book1 = new Book(req.body);
    book1.save(function (err, doc) {
        Book.find(function (err, data) {
            res.json(data)
        });
    });
});

clientApp.js
$scope.addBook = function (book) {
        $http.post("/book", book)
        .success(function (response) {
            $scope.books = response;
        });
    }

Description

When the + button is clicked, the addBook handler is invoked which generates a http post using $http.post which looks for a match in the form of /book.
A new book instance is created with its contents as the req.body which is done using var book1 = new Book(req.body). book1.save is used to commit the data into the database and after it is saved, we use Book.find to display all the records after the commit. If a book is selected, the name of the book is displayed.

The select handler is invoked when the edit button is clicked which retrieves the selectedIndex.
